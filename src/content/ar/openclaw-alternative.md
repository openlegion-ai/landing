# بديل OpenClaw: وكلاء ذكاء اصطناعي آمنون مع OpenLegion

إذا كنت تبحث عن **بديل لـ OpenClaw**، فمن المرجح أنك واجهت إحدى نقاط الاحتكاك التالية: اشتراط مقبس Docker يمنح وصولاً أكبر مما ينبغي للمضيف لوضعك الأمني، أو تحتاج إلى عزل بيانات اعتماد يتجاوز إخفاء الأسرار داخل العملية، أو تريد ضوابط تكلفة لكل وكيل لمنع الإنفاق الجامح، أو تحتاج إلى تنسيق أسطول متعدد الوكلاء بدلاً من وكيل ترميز واحد.

OpenLegion هو [إطار وكلاء ذكاء اصطناعي](/learn/ai-agent-platform) متوفر المصدر مبني للفرق التي تحتاج إلى أمان وحوكمة بمستوى إنتاج. أحضر مفاتيح LLM API الخاصة بك (أو استخدم أرصدة مُدارة). لا هامش ربح على استخدام النموذج بمفاتيحك.

<!-- SCHEMA: DefinitionBlock -->

> **لماذا البحث عن بديل لـ OpenClaw؟**
> تبحث الفرق عن بدائل لـ OpenClaw عندما تحتاج إلى إعدادات أمان افتراضية أكثر صرامة (عزل حاوية إلزامي دون تركيب مقبس Docker)، أو إدارة بيانات اعتماد لا يرى الوكلاء فيها مفاتيح API الخام أبداً، أو فرض ميزانية لكل وكيل، أو نموذج تنسيق أسطول منظَّم لعمليات متعددة الوكلاء قابلة للتدقيق.

## الخلاصة

- **عزل الحاويات** — كل وكيل في حاوية Docker خاصة به. لا تركيب لمقبس Docker. غير الجذر، no-new-privileges، حدود موارد قابلة للتهيئة.
- **بيانات اعتماد ممرّرة عبر وكيل الخزنة** — خزنة بيانات اعتماد مع مقابض `$CRED{name}`. لا يرى الوكلاء المفاتيح الخام أبداً؛ يحقن الوكيل بيانات الاعتماد على طبقة الشبكة.
- **ضوابط ميزانية لكل وكيل** — حدود يومية وشهرية مع قطع صارم. لا فواتير مفاجئة.
- **تنسيق الأسطول** — نموذج الأسطول: سبورة (SQLite CAS) + نشر/اشتراك + تسليم منظَّم. 13 قالباً جاهزاً في YAML.
- **متعدد القنوات** — CLI وTelegram وDiscord وSlack وWhatsApp (نص فقط؛ يتطلب الإنتاج `WHATSAPP_APP_SECRET`) — بالإضافة إلى نقاط نهاية webhook للتكاملات الخارجية. ليس مجرد واجهة ويب.
- **بدون خدمات خارجية** — Python + SQLite + Docker. لا Redis، ولا Kubernetes، ولا LangChain.

## مقارنة سريعة

| القدرة | OpenClaw | OpenLegion |
|---|---|---|
| **عزل الوكلاء** | على مستوى العملية | حاوية Docker لكل وكيل، لا مقبس Docker، غير الجذر |
| **معالجة بيانات الاعتماد** | Secret Registry — الأسرار متاحة لعملية الوكيل | وكيل خزنة — الوكلاء لا يرون المفاتيح الخام أبداً |
| **ضوابط التكلفة** | لا توجد | ميزانيات يومية/شهرية لكل وكيل مع قطع صارم |
| **التنسيق** | قائم على الأحداث، قائم على SDK | نموذج الأسطول — سبورة + نشر/اشتراك + تسليم (بدون وكيل رئيس تنفيذي) |
| **متعدد الوكلاء** | وكيل واحد أساسي، SDK يدعم تعدد الوكلاء | نموذج أسطول أصلي مع تنسيق سبورة، ونشر/اشتراك، وبروتوكول تسليم منظَّم |
| **قنوات النشر** | واجهة ويب، CLI | CLI وTelegram وDiscord وSlack وWhatsApp + webhooks |
| **الاعتماديات** | Python، Docker (+ النظام البيئي) | Python، SQLite، Docker (بدون خدمات خارجية) |
| **دعم LLM** | متوافق مع LiteLLM | 100+ عبر LiteLLM |
| **المجتمع** | 200 ألف+ نجمة GitHub | مشروع جديد، فريق صغير |
| **الأفضل لـ** | تطوير برمجي مدفوع بالذكاء الاصطناعي | عمليات أسطول وكلاء متعددة آمنة |

للحصول على تفصيل أعمق لاختلافات البنية، راجع [مقارنة OpenLegion مقابل OpenClaw](/comparison/openclaw) الكاملة.

## لماذا تنتقل الفرق

**تشير فرق الأمان** إلى متطلب مقبس Docker. تركيب `/var/run/docker.sock` في حاوية الوكيل هو فعلياً وصول مكافئ للجذر إلى المضيف. يدير Mesh Host في OpenLegion الحاويات عبر Docker API من منطقة موثوقة — لا تملك حاوية الوكيل وصولاً إلى مقبس Docker.

**الفرق التي تتعامل مع بيانات اعتماد الإنتاج** تحتاج إلى أكثر من إخفاء الأسرار. يخفي Secret Registry في OpenClaw الأسرار في المخرجات، لكن الأسرار لا تزال موجودة في ذاكرة عملية الوكيل. يبقي وكيل الخزنة في OpenLegion الأسرار خارج حاوية الوكيل تماماً — يرسل الوكيل طلباً، ويحقن الوكيل بيانات الاعتماد، ويستلم الوكيل النتيجة. حتى وكيل مخترق بالكامل لا يمكنه استخراج بيانات الاعتماد.

**الفرق التي تحرق الميزانية على حلقات الوكلاء** تحتاج إلى حدود صارمة. بدون ضوابط تكلفة مدمجة، يمكن لحلقة تكرارية أو وكيل سيء التهيئة أن يستهلك مئات الدولارات قبل التدخل اليدوي. تفرض ضوابط الميزانية لكل وكيل في OpenLegion الحدود على [طبقة التنسيق](/learn/ai-agent-orchestration) مع قطع تلقائي.

**الفرق التي تنشر إلى قنوات تواجه العملاء** تحتاج إلى أكثر من واجهة ويب. ينشر OpenLegion الوكلاء إلى CLI وTelegram وDiscord وSlack وWhatsApp — بالإضافة إلى نقاط نهاية webhook للتكاملات الخارجية — عبر رموز قنوات مهيّأة بالبيئة.

## البدء

```bash
git clone https://github.com/openlegion-ai/openlegion.git
cd openlegion && ./install.sh
openlegion start   # إعداد مضمّن عند التشغيل الأول، ثم تنشر الوكلاء في حاويات معزولة
```

ثلاثة أوامر؛ يستغرق بناء صور Docker عند التشغيل الأول دقائق قليلة (صورة وكيل واحدة، وصورة خدمة متصفح واحدة). يتطلب Python 3.10+ وDocker.

## الأسئلة المتكررة

### ما أفضل بديل لـ OpenClaw؟

للفرق التي اهتمامها الأساسي هو الأمان والحوكمة، OpenLegion هو البديل الأكثر مباشرة لـ OpenClaw. يوفّر قدرات تفتقر إليها OpenClaw: عزل حاوية إلزامي دون تركيب مقبس Docker، وبيانات اعتماد ممرّرة عبر وكيل الخزنة، وفرض ميزانية لكل وكيل، ونموذج تنسيق أسطول (سبورة + نشر/اشتراك + تسليم). للفرق التي تركّز على مرونة سير العمل ذي الحالة، LangGraph بديل قوي آخر. راجع [مقارنة أطر وكلاء الذكاء الاصطناعي](/learn/ai-agent-frameworks) الكاملة.

### لماذا اختيار بديل مُدار لـ OpenClaw؟

يتعامل البديل المُدار لـ OpenClaw مع طبقة الأمان التشغيلي التي تتطلب منك عمليات نشر OpenClaw المستضافة ذاتياً بناءها: تشديد الحاويات، وتخزين بيانات الاعتماد، وتتبع التكلفة، والنشر متعدد القنوات. يوفّر OpenLegion هذه كميزات إطار مدمجة. يقلل هذا من استثمار DevOps المطلوب للانتقال من النموذج الأولي إلى الإنتاج مع تحسين الوضع الأمني لأسطول الوكلاء الخاص بك.

### OpenClaw مقابل OpenLegion: أيهما يجب أن أستخدم؟

استخدم OpenClaw إذا كنت بحاجة إلى وكيل ترميز ذكاء اصطناعي متخصص، أو تريد أكبر مجتمع مفتوح المصدر، أو تعطي الأولوية لأقصى مرونة في الاستضافة الذاتية. استخدم OpenLegion إذا كنت بحاجة إلى عزل بيانات الاعتماد (الوكلاء لا يرون المفاتيح أبداً)، أو ضوابط ميزانية لكل وكيل، أو نموذج تنسيق أسطول منظَّم، أو كنت تنشر أساطيل وكلاء متعددة عبر قنوات تواجه العملاء. للمقارنة التفصيلية، راجع [OpenLegion مقابل OpenClaw](/comparison/openclaw).

### هل يتطلب OpenLegion مفاتيح LLM API الخاصة بي؟

يدعم OpenLegion BYOK (أحضر مفاتيحك الخاصة). يمكنك تقديم مفاتيح API الخاصة بك من أي مزوّد LLM — OpenAI أو Anthropic أو Google أو Mistral أو 100+ آخرين عبر LiteLLM. تُحفظ مفاتيحك في خزنة بيانات الاعتماد في Mesh Host وتُحقن عبر وكيل الخزنة. لا يرى الوكلاء المفاتيح الخام أبداً. تدفع للمزوّدين مباشرة بأسعارهم المعلنة دون هامش ربح. تقدّم الاستضافة المُدارة أيضاً أرصدة LLM مدفوعة مسبقاً للتسهيل.

### هل يمكنني الاستضافة الذاتية بدلاً من استخدام OpenLegion المُستضاف؟

نعم. OpenLegion متوفر المصدر تحت ترخيص PolyForm Perimeter License 1.0.1. تتطلب الاستضافة الذاتية Python 3.10+ وDocker. عملية التثبيت هي `git clone && ./install.sh && openlegion start`؛ يستغرق بناء صور Docker عند التشغيل الأول دقائق قليلة. لا تتطلب خدمات خارجية — لا Redis، ولا Kubernetes، ولا خدمات سحابية. يعمل على جهاز واحد. يتوفر أيضاً خيار مستضاف للفرق التي تفضّل البنية التحتية المُدارة.

### ما مدى صعوبة الترحيل من OpenClaw إلى OpenLegion؟

يستخدم كلا المشروعين Python لتعريفات الوكلاء وتوجيه نماذج متوافق مع LiteLLM، لذا تنتقل تهيئات LLM مباشرة. تتطلب تكاملات الأدوات التكيّف مع مصفوفة أذونات OpenLegion، وستحدّد أساطيل الوكلاء عبر قوالب YAML الخاصة بـ OpenLegion. ترحيل بيانات الاعتماد هو تهيئة خزنة لمرة واحدة. المقايضة الرئيسية: تكتسب عزلاً إلزامياً، وبيانات اعتماد ممرّرة عبر وكيل الخزنة، وضوابط ميزانية؛ تفقد قدرات الترميز المتخصصة في OpenClaw ونظامها البيئي المجتمعي الكبير.
