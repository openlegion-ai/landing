# OpenLegion مقابل LangGraph: إطار يضع الأمان أولاً مقابل معيار التنسيق

LangGraph هو إطار تنسيق الوكلاء الأكثر تبنياً على نطاق واسع في الإنتاج. مبني من قبل فريق LangChain، لديه نحو 25,200 نجمة GitHub، و6.17 مليون تنزيل شهري من PyPI، ووصل إلى الإطلاق العام 1.0 في 22 أكتوبر 2025 — أول إطار وكلاء رئيسي يصل إلى إصدار مستقر. تظهر عمليات النشر للمؤسسات في Uber وLinkedIn وKlarna وReplit التبني الفعلي على نطاق واسع.

OpenLegion هو [إطار وكلاء ذكاء اصطناعي](/learn/ai-agent-platform) يضع الأمان أولاً مع عزل حاوية Docker إلزامي، وإدارة بيانات اعتماد عبر وكيل الخزنة، وفرض ميزانية لكل وكيل، وتنسيق نموذج الأسطول (سبورة + نشر/اشتراك + تسليم).

يمثّل LangGraph وOpenLegion إجابتين مختلفتين لنفس السؤال: كيف ينبغي تنسيق سير عمل الوكلاء؟ يقول LangGraph: امنح المطوّرين بدائيات الرسم البياني بأقصى مرونة. يقول OpenLegion: امنح المطوّرين تنسيق نموذج أسطول قابلاً للتدقيق بأقصى أمان. كلاهما صحيح — الاختيار الصحيح يعتمد على ما إذا كان عنق الزجاجة هو تعقيد التنسيق أم مخاطر الأمان.

> **ما الفرق بين OpenLegion وLangGraph؟**
> LangGraph هو إطار تنسيق قائم على الرسم البياني لبناء وكلاء ذكاء اصطناعي ذوي حالة طويلة الأمد برسوم بيانية موجَّهة (بما في ذلك الدورات)، وتنفيذ نقاط فحص/إعادة تشغيل دائم، وتكامل عميق مع نظام LangChain البيئي. OpenLegion هو إطار وكلاء ذكاء اصطناعي يضع الأمان أولاً مع عزل حاوية Docker إلزامي، وإدارة بيانات اعتماد عبر وكيل الخزنة حيث لا ترى الوكلاء مفاتيح API أبداً، وفرض ميزانية لكل وكيل، وتنسيق نموذج الأسطول (سبورة + نشر/اشتراك + تسليم). يمنحك LangGraph أقصى مرونة في التنسيق؛ يمنحك OpenLegion أقصى سلامة إنتاجية.

## الخلاصة

| البُعد | OpenLegion | LangGraph |
|---|---|---|
| **التركيز الأساسي** | بنية تحتية لأمان الإنتاج | تنسيق ذو حالة قائم على الرسم البياني |
| **البنية** | نموذج ثقة رباعي المناطق (المستخدم ← Mesh Host ← حاويات الوكلاء، بالإضافة إلى مشغّل أو داخلي) | StateGraph مع حالة مكتوبة، عقد، حواف شرطية، نقاط فحص |
| **عزل الوكلاء** | حاوية Docker لكل وكيل، غير الجذر، no-new-privileges | لا عزل مدمج؛ صندوق رمل Pyodide/WASM لتنفيذ الشيفرة فقط |
| **أمان بيانات الاعتماد** | وكيل الخزنة — لا ترى الوكلاء المفاتيح أبداً | لا نظام مدمج؛ يعتمد على متغيرات البيئة أو الخزائن الخارجية |
| **ضوابط الميزانية** | قطع يومي/شهري صارم لكل وكيل | لا توجد أصلية؛ LangSmith يوفّر تتبع التكلفة فقط |
| **التنسيق** | تنسيق نموذج الأسطول — سبورة + نشر/اشتراك + تسليم (بدون وكيل رئيس تنفيذي) | رسوم بيانية موجَّهة مع دورات، وحواف شرطية، وتوجيه قائم على الأوامر |
| **التنفيذ الدائم** | حالة المهمة محفوظة في SQLite | قائم على نقاط الفحص (PostgreSQL/SQLite)، يبقى عبر إعادة التشغيل، سفر عبر الزمن |

| **تعدد الوكلاء** | قوالب أسطول مع ACLs لكل وكيل | مشرف، سرب، رسم بياني للرسوم البيانية (تركيب رسم بياني فرعي) |

| **الرصد** | لوحة تحكم مدمجة | LangSmith (تتبع، تقييم، مراقبة) |

| **تنزيلات PyPI** | قبل الإصدار | ~6.17 مليون/شهر |
| **ثغرات CVE المعروفة** | 0 | 4 حرجة في نظام LangChain البيئي (حتى CVSS 9.3) |

| **التسعير** | مفاتيح API خاصة بك، 19 دولاراً/شهر مستضاف | مجاني (MIT)؛ LangSmith Plus 39 دولاراً/مقعد/شهر للمصادقة/RBAC |

## اختر LangGraph إذا...

**تحتاج إلى سير عمل ذي حالة معقّد مع دورات.** يتعامل نموذج الرسم البياني في LangGraph مع التفرّع والحلقات والتوجيه الشرطي الذي لا يمكن لتنسيق نموذج الأسطول التعبير عنه. إذا كان سير عمل وكيلك يتطلب تفرّعاً ديناميكياً بناءً على نتائج وسيطة — وكيل بحث يدور حتى يتم استيفاء حدود الجودة، أو مشرف يعيد توجيه المهام الفاشلة — فإن LangGraph مصمَّم لهذا الغرض.

**تحتاج إلى تنفيذ دائم مع نقاط فحص/إعادة تشغيل.** يتيح نظام نقاط الفحص في LangGraph (مدعوم بـ PostgreSQL أو SQLite) سير العمل البقاء عبر إعادة تشغيل الخادم، ويتيح تصحيح السفر عبر الزمن من أي حالة تاريخية، ويدعم التفرّع من أي نقطة فحص. هذه قدرة ناضجة لا يضاهيها أي إطار آخر.

**تريد نظام LangChain البيئي.** يتكامل LangGraph مع LangSmith لرصد الإنتاج، و700+ تكامل من LangChain، وأوسع مجتمع مطوّري وكلاء. تظهر عمليات النشر الإنتاجية في Uber وLinkedIn وKlarna وReplit تبنّي المؤسسات.

**لديك بالفعل بنية تحتية أمنية.** إذا كانت مؤسستك تشغّل مدير أسرار، وتنسيق حاويات، وأمان شبكة، فإن مرونة LangGraph تتيح لك إضافة سير عمل الوكلاء على البنية التحتية الموجودة دون تكرار بدائيات الأمان.

**تريد إطار الوكلاء الوحيد بإطلاق عام 1.0.** LangGraph 1.0 (أكتوبر 2025) هو إطار الوكلاء الرئيسي الوحيد بإصدار مستقر. للفرق التي تتطلب ضمانات استقرار API، هذا يهم.

## اختر OpenLegion إذا...

**أمان بيانات الاعتماد متطلب صارم.** LangGraph ليس لديه إدارة بيانات اعتماد مدمجة وله تاريخ من ثغرات التسلسل التي قد تكشف الأسرار. أظهرت ثغرة حقن التسلسل (CVSS 9.3، ديسمبر 2025) أن التلاعب بنقاط الفحص يمكن أن يستخرج الأسرار وينفّذ شيفرة تعسفية. يوفّر وكيل الخزنة في OpenLegion حماية معمارية — لا ترى الوكلاء مفاتيح API أبداً، حتى لو اختُرقت عملية الوكيل.

**تحتاج إلى فرض ميزانية لكل وكيل.** يوفّر LangGraph تتبع التكلفة عبر LangSmith لكن لا توجد آلية لإيقاف وكيل تلقائياً يتجاوز عتبة الإنفاق. وكيل عالق في حلقة استدلال سيستمر في تراكم التكاليف حتى الإنهاء اليدوي. يفرض OpenLegion قطعاً صارماً لكل وكيل، ولكل يوم، ولكل شهر — عندما تستنفد الميزانية، يتوقف الوكيل.

**تريد الأمان مدمجاً، وليس مُركَّباً.** تظهر ثغرات CVE الأربع الحرجة في نظام LangChain البيئي في 18 شهراً التحدي في إضافة الأمان إلى إطار غير مصمَّم له. أُضيف تشفير AES لنقاط الفحص وصندوق رمل Pyodide بأثر رجعي. كان نموذج الثقة رباعي المناطق في OpenLegion (بالإضافة إلى مستوى مشغّل أو داخلي) هو البنية البدائية.

**تحتاج إلى تنسيق نموذج أسطول قابل للتدقيق.** يمكن مراجعة قوالب الأسطول وACLs، والتحكم في إصدارها، وتدقيقها للامتثال قبل أن ينفّذ أي وكيل. التنسيق محدود بكشف حلقة الأدوات لكل وكيل (تحذير@2، حظر@4، إنهاء@9). من الأصعب تدقيق سير العمل القائم على الرسم البياني مع التوجيه الديناميكي ثابتاً، وتقدّم الدورات إمكانية حلقات لا نهائية بدون كشف محدود.

**تريد صفر اعتماديات خارجية.** يعمل OpenLegion على Python + SQLite + Docker. يتطلب LangGraph نظام LangChain البيئي وعادةً LangSmith (39 دولاراً/مقعد/شهر على Plus) لميزات الإنتاج مثل المصادقة وRBAC.

## مقارنة نموذج الأمان

### أين تعيش الأسرار

**LangGraph** ليس لديه إدارة أسرار أو بيانات اعتماد مدمجة. يستخدم المطوّرون عادةً متغيرات البيئة، أو ملفات `.env`، أو يدمجون حلول خزائن خارجية (HashiCorp Vault، AWS Secrets Manager). يعني هذا أن بيانات الاعتماد موجودة في بيئة عملية الوكيل — يمكن الوصول إليها بأي شيفرة تعمل في تلك العملية. أظهرت ثغرة حقن التسلسل أن بيانات نقاط الفحص يمكن التلاعب بها لاستخراج متغيرات البيئة بما في ذلك مفاتيح API.

**OpenLegion** يخزّن بيانات الاعتماد في خزنة لا يمكن الوصول إليها إلا عبر وكيل. تجري الوكلاء استدعاءات API عبر وكيل الخزنة؛ تُحقن بيانات الاعتماد على مستوى الشبكة. لا توجد متغيرات بيئة بمفاتيح API، ولا ملفات `.env`، ولا كائنات سرية في ذاكرة الوكيل. حتى لو اختُرقت بيانات نقاط الفحص أو حالة الوكيل، لا توجد بيانات اعتماد للاستخراج.

### نموذج العزل

**LangGraph** يعمل كمكتبة Python داخل عملية تطبيقك. لا يوجد عزل وكلاء مدمج — تتشارك جميع الوكلاء والأدوات وسير العمل نفس مساحة العملية. صندوق رمل Pyodide/WebAssembly (المُضاف في مايو 2025) يعزل تنفيذ الشيفرة تحديداً، لكن منطق الوكيل نفسه يعمل في عملية المضيف. المصادقة وRBAC متاحان فقط في مستويات LangSmith Plus وEnterprise.

**OpenLegion** يستخدم عزل حاوية Docker لكل وكيل. يعمل كل وكيل في حاوية منفصلة مع تنفيذ غير الجذر، ودون مقبس Docker، وno-new-privileges، وحدود موارد لكل حاوية. لا يمكن للوكلاء الوصول إلى وكلاء آخرين، أو نظام المضيف، أو مخازن بيانات الاعتماد. هذا عزل على مستوى نظام التشغيل تفرضه مساحات أسماء Linux وcgroups.

### سجل CVE

**نظام LangChain البيئي** راكم ثغرات CVE حرجة متعددة تؤثر على مستخدمي LangGraph:

- **حقن prompt hub (CVSS 8.8، أكتوبر 2024):** قد تسرق إدخالات prompt hub الخبيثة مفاتيح API.
- **RCE عبر إلغاء التسلسل (حرجة، نوفمبر 2025):** تنفيذ شيفرة عن بُعد عبر تسلسل نقاط الفحص.
- **حقن التسلسل (CVSS 9.3، ديسمبر 2025):** حقن تسلسل يستخرج الأسرار وينفّذ شيفرة تعسفية.
- **ثغرات نقاط فحص إضافية** عُولِجَت بتشفير AES (يناير 2026).

**OpenLegion** ليس لديه ثغرات CVE مبلّغ عنها اعتباراً من v0.1.0. تعني بنية وكيل الخزنة لديه أنه لا توجد بيانات اعتماد في حالة الوكيل لاستخراجها عبر هجمات التسلسل.

### ضوابط الميزانية

**LangGraph** يوفّر تتبع التكلفة والرصد عبر LangSmith لكن لا توجد آلية لفرض حدود الإنفاق. الوكيل في حلقة الاستدلال يستمر في تراكم التكاليف.

**OpenLegion** يفرض حدود ميزانية يومية وشهرية لكل وكيل مع قطع تلقائي صارم.

## نظام LangGraph البيئي: ما يفعله بشكل أفضل

### بدائيات التنسيق هي الأفضل في فئتها

تجريد StateGraph في LangGraph هو أكثر نماذج تنسيق الوكلاء تعبيراً متاحة. مخططات الحالة المكتوبة، والحواف الشرطية، والتوجيه القائم على الأوامر، وتركيب الرسم البياني الفرعي، وتفريع map-reduce تتيح لك نمذجة سير عمل لا تستطيع أطر أخرى التعبير عنه. يوفّر بدائي `interrupt` للإنسان في الحلقة، مدمجاً مع السفر عبر الزمن القائم على نقاط الفحص، قدرات تصحيح وإعادة تشغيل لا يضاهيها منافس.

### التنفيذ الدائم فريد حقاً

تبقى سير عمل LangGraph عبر إعادة تشغيل الخادم. يمكنك إعادة التشغيل من أي نقطة فحص، والتفرّع من حالات تاريخية، وتصحيح الأخطاء بالتنقّل عبر تسلسل انتقالات الحالة الدقيق. للوكلاء طويلي الأمد (مهام البحث التي تستغرق ساعات، وسير عمل الموافقة التي تمتد لأيام)، هذه المتانة أساسية.

### تبنّي المؤسسات يصادق على البنية

عمليات النشر في Uber وLinkedIn وKlarna وReplit ليست نظرية. هذه أنظمة إنتاج تتعامل مع أعباء عمل حقيقية. يوفّر هذا التبنّي ثقة في الاستقرار والأداء والدعم طويل الأمد التي لا تستطيع الأطر قبل الإصدار تقديمها.

### منصة إنتاج LangSmith

يضيف LangSmith التتبع والتقييم والمراقبة و(في مستويات Plus/Enterprise) المصادقة وRBAC. إطار التقييم لاختبار سلوك الوكلاء قيّم بشكل خاص — الاختبار المنهجي لمخرجات الوكلاء قدرة تفتقر إليها معظم الأطر تماماً.

### مزالق الإنتاج الشائعة

**الأمان يتطلب بنية تحتية خارجية.** LangGraph لا يقدّم إدارة بيانات اعتماد، أو عزل وكلاء، أو أمان شبكة. يجب على عمليات النشر الإنتاجية إضافة هذه فوقه باستخدام أدوات خارجية (Kubernetes، HashiCorp Vault، سياسات الشبكة). تواجه الفرق بدون بنية تحتية أمنية موجودة إعداداً كبيراً.

**نمط ثغرة التسلسل.** تتعلق ثلاث من أصل أربع ثغرات CVE بالتسلسل/إلغاء التسلسل — فئة ثغرات متكررة في الأنظمة القائمة على نقاط الفحص. يعالج إصلاح تشفير AES النواقل المعروفة لكن النمط المعماري (تسلسل حالة الوكيل بما في ذلك مخرجات الأدوات) يبقى مساحة سطح.

**تكلفة LangSmith على نطاق واسع.** 39 دولاراً/مقعد/شهر لـ Plus (مطلوب للمصادقة وRBAC) يتدرّج خطياً. تواجه الفرق الكبيرة تكاليف منصة هامة قبل أي إنفاق LLM.

**تكلفة التعقيد.** تأتي مرونة LangGraph مع منحنى تعلّم. طبقة التجريد (StateGraph، مخططات TypedDict، الحواف الشرطية، توجيه Command، تسلسل نقاط الفحص، تركيب الرسم البياني الفرعي) قوية لكنها تتطلب استثماراً كبيراً من المطوّر.

### ما يغطيه OpenLegion بشكل مختلف

يشمل OpenLegion بدائيات أمان يتطلب LangGraph منك الحصول عليها خارجياً: يستبدل وكيل الخزنة تكامل HashiCorp Vault، يستبدل عزل حاوية Docker عزل pod في Kubernetes، تستبدل ميزانيات كل وكيل المراقبة اليدوية للتكلفة، يستبدل تنسيق نموذج الأسطول سير العمل القائم على الرسم البياني بقابلية تدقيق ثابتة، وتستبدل صفر اعتماديات خارجية مكدّس نظام LangChain البيئي.

## مقايضات الاستضافة مقابل الاستضافة الذاتية

**LangGraph** هو مكتبة Python تستضيفها بنفسك. يوفّر LangSmith منصة سحابية اختيارية للرصد والمصادقة وRBAC. الاستضافة الذاتية لـ LangSmith Enterprise متاحة بتسعير مؤسسات. يمنح ترخيص MIT مرونة نشر كاملة.

**OpenLegion** يتطلب Python وSQLite وDocker. تقدّم المنصة المستضافة (قريباً) مثيلات VPS لكل مستخدم بـ 19 دولاراً/شهر مع مفاتيح API خاصة بك. النشر المستضاف ذاتياً مكتفٍ ذاتياً بالكامل بصفر اعتماديات خدمات خارجية.

## لمن هو

**LangGraph** لفرق الهندسة التي تبني سير عمل وكلاء ذو حالة معقّد ويتطلب تحكماً دقيقاً في تدفق التنفيذ، ونقاط فحص/إعادة تشغيل دائم، وتكامل عميق مع النظام البيئي. المستخدم المثالي هو مهندس backend مرتاح للتجريدات القائمة على الرسم البياني ولديه وصول إلى بنية تحتية أمنية موجودة (مديرو الأسرار، تنسيق الحاويات، سياسات الشبكة) ويقدّر مرونة التنسيق على الأمان المدمج.

**OpenLegion** للفرق التي تنشر أساطيل وكلاء في بيئات حيث أمان بيانات الاعتماد، والتحكم في التكلفة، والقابلية للتدقيق متطلبات صارمة — والذين يريدون هذه القدرات مدمجة في الإطار بدلاً من تجميعها من أدوات خارجية. المستخدم المثالي يحتاج إلى إظهار وضع أمني لمراجعي الامتثال ولا يمكنه المخاطرة بتعرّض بيانات الاعتماد أو التكاليف غير المنضبطة.

## المقايضة الصادقة

LangGraph لديه قوة التنسيق، ونضج الإنتاج (1.0 GA)، وتبنّي المؤسسات، واتساع النظام البيئي. يتعامل نموذجه القائم على الرسم البياني مع سير العمل الذي لا يستطيع تنسيق نموذج الأسطول التعبير عنه.

OpenLegion لديه بنية الأمان، وحماية بيانات الاعتماد، وحوكمة التكلفة المدمجة. تنسيق نموذج الأسطول لديه أقل تعبيراً من رسوم بيانية LangGraph لكنها توفّر قابلية تدقيق ثابتة وضمانات سلامة هيكلية.

إذا كان عنق الزجاجة هو تعقيد التنسيق، اختر LangGraph. إذا كان عنق الزجاجة هو مخاطر الأمان، اختر OpenLegion. تستخدم بعض الفرق كليهما: LangGraph لسير العمل الداخلي المعقّد، وOpenLegion للوكلاء الذين يواجهون الخارج ويتعاملون مع بيانات اعتماد حساسة.

للمشهد الكامل، راجع [مقارنة أطر وكلاء الذكاء الاصطناعي](/learn/ai-agent-frameworks).

## الأسئلة المتكررة

### ما هو LangGraph؟

LangGraph هو إطار تنسيق وكلاء قائم على الرسم البياني مبني من قبل فريق LangChain. بنحو 25,200 نجمة GitHub و6.17 مليون تنزيل شهري من PyPI، ينمذج سير عمل الوكلاء كرسوم بيانية موجَّهة بحالة مكتوبة، وحواف شرطية، وتنفيذ نقاط فحص/إعادة تشغيل دائم. وصل إلى الإطلاق العام 1.0 في 22 أكتوبر 2025 ومنشور في Uber وLinkedIn وKlarna وReplit.

### OpenLegion مقابل LangGraph: ما الفرق؟

LangGraph هو إطار تنسيق قائم على الرسم البياني مُحسَّن لسير العمل ذي الحالة المعقّد مع الدورات، ونقاط الفحص/إعادة التشغيل، وتكامل نظام LangChain البيئي. OpenLegion هو إطار يضع الأمان أولاً مع عزل حاوية Docker، وبيانات اعتماد عبر وكيل الخزنة (لا ترى الوكلاء المفاتيح أبداً)، وميزانيات لكل وكيل، وتنسيق نموذج الأسطول (سبورة + نشر/اشتراك + تسليم). يقدّم LangGraph مرونة تنسيق أكثر؛ يقدّم OpenLegion ضمانات أمان أقوى.

### هل OpenLegion بديل لـ LangGraph؟

نعم. يخدم OpenLegion كبديل لـ LangGraph للفرق التي يكون متطلبها الأساسي هو الأمان المدمج بدلاً من مرونة التنسيق. يوفّر قدرات يفتقر إليها LangGraph أصلياً: عزل حاوية إلزامي، وإدارة بيانات اعتماد عبر وكيل الخزنة، وفرض ميزانية لكل وكيل، وتنسيق نموذج أسطول قابل للتدقيق. لا يكرّر دورات LangGraph القائمة على الرسم البياني، أو نقاط الفحص/إعادة التشغيل الدائم، أو تكامل نظام LangChain البيئي.

### كيف تقارَن معالجة بيانات الاعتماد بين OpenLegion وLangGraph؟

LangGraph ليس لديه إدارة بيانات اعتماد مدمجة — يستخدم المطوّرون متغيرات البيئة أو الخزائن الخارجية. تتعلق ثلاث من أصل أربع ثغرات CVE بثغرات تسلسل قد تكشف الأسرار. يوجّه وكيل الخزنة في OpenLegion استدعاءات API عبر وكيل يحقن بيانات الاعتماد على مستوى الشبكة. لا تحمل الوكلاء أبداً مفاتيح بأي شكل، مما يجعل سرقة بيانات الاعتماد القائمة على التسلسل مستحيلة هيكلياً.

### أيهما أفضل لوكلاء الذكاء الاصطناعي للإنتاج؟

LangGraph لديه نضج إنتاج أقوى (1.0 GA، تبنّي المؤسسات). OpenLegion لديه أمان إنتاج أقوى (وكيل الخزنة، عزل الحاويات، ميزانيات لكل وكيل). لسير العمل الداخلي المعقّد مع بنية تحتية أمنية موجودة، LangGraph. لأساطيل الوكلاء التي تتعامل مع بيانات اعتماد حساسة حيث الأمان المدمج مطلوب، OpenLegion.

### هل لدى LangGraph ضوابط تكلفة لكل وكيل؟

يوفّر LangGraph تتبع التكلفة عبر LangSmith لكن لا توجد آلية لفرض حدود الإنفاق أو إيقاف الوكلاء تلقائياً الذين يتجاوزون الميزانيات. يفرض OpenLegion حدوداً يومية وشهرية لكل وكيل مع قطع تلقائي صارم.

### هل LangGraph آمن لعمليات النشر الإنتاجية؟

راكم نظام LangChain البيئي 4 ثغرات CVE حرجة (حتى CVSS 9.3) بما في ذلك حقن التسلسل وRCE التي تؤثر على مستخدمي LangGraph. استجاب الفريق بتشفير AES لنقاط الفحص وصندوق رمل Pyodide. للفرق التي يكون فيها الأمان الأولوية القصوى، توفّر بنية OpenLegion على مستوى البنية ضمانات افتراضية أقوى. للفرق ذات البنية التحتية الأمنية الموجودة، تسمح مرونة LangGraph بإضافة الأمان فوقه.

### هل يمكنني استخدام LangGraph وOpenLegion معاً؟

نعم. تستخدم بعض الفرق LangGraph للتنسيق الداخلي المعقّد وOpenLegion للوكلاء الذين يواجهون الخارج ويتعاملون مع بيانات اعتماد حساسة. يعني دعم خادم أدوات MCP في OpenLegion أن وكلاء LangGraph يمكنهم استهلاك أدوات يديرها OpenLegion.
