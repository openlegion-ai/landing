# منصة وكلاء الذكاء الاصطناعي المبنية للإنتاج

تبدأ معظم الفرق بإطار. تربط عقد LangGraph أو فرق CrewAI ببعضها، وتجعل عرضاً توضيحياً يعمل، ثم تصطدم بحائط: من يدير الحاويات؟ أين تذهب مفاتيح API؟ ما الذي يمنع وكيلاً مارقاً من حرق 500 دولار من الرموز بين عشية وضحاها؟

**منصة وكلاء الذكاء الاصطناعي** تجيب على هذه الأسئلة قبل أن تكتب وكيلك الأول. OpenLegion هي منصة وكلاء ذكاء اصطناعي مُدارة تقدّم عزل الحاويات، وبيانات اعتماد ممرّرة عبر وكيل الخزنة، وضوابط ميزانية لكل وكيل، وتنسيق نموذج الأسطول (سبورة + نشر/اشتراك + تسليم) — كل ذلك مفعّل افتراضياً. أحضر مفاتيح LLM API الخاصة بك. لا هامش ربح على استخدام النموذج.

> **ما هي منصة وكلاء الذكاء الاصطناعي؟**
> منصة وكلاء الذكاء الاصطناعي هي بنية تحتية مُدارة لنشر وتنسيق وحوكمة وكلاء ذكاء اصطناعي مستقلين في الإنتاج. على عكس الأطر الخام، تتعامل المنصة مع العزل، وإدارة بيانات الاعتماد، وضوابط التكلفة، والرصد بحيث تطلق الفرق الوكلاء دون بناء DevOps من الصفر.

## الخلاصة

- **منصة، لا إطار** — تدير OpenLegion الحاويات، وبيانات الاعتماد، والميزانيات، والشبكات. أنت تدير منطق الوكيل.
- **بيانات اعتماد ممرّرة عبر وكيل الخزنة** — تنفّذ الوكلاء استدعاءات API عبر وكيل خزنة. لا ترى المفاتيح الخام أبداً.
- **عزل الحاوية لكل وكيل** — يعمل كل وكيل في حاوية Docker خاصة به مع حدود موارد قابلة للتهيئة (384 ميغابايت ذاكرة / 0.15 CPU افتراضياً)، وتنفيذ غير الجذر، ولا نظام ملفات مشترك.
- **فرض ميزانية لكل وكيل** — حدد حدود رموز يومية وشهرية مع قطع تلقائي صارم. لا فواتير مفاجئة.
- **مفاتيح API خاصة بك** — اتصل بأي مزوّد LLM عبر LiteLLM (100+ مدعوم). تدفع للمزوّدين مباشرة بأسعارهم المعلنة.
- **تنسيق نموذج أسطول قابل للتدقيق** — تنسيق نموذج الأسطول (سبورة + نشر/اشتراك + تسليم) لتوجيه المهام. لا "وكيل رئيس تنفيذي" يتخذ قرارات غامضة.
- **قابلية التوسع المتوافقة مع MCP** — اتصل بأي خادم أداة MCP (قواعد بيانات، أنظمة ملفات، APIs) إلى جانب 50+ مهارة مدمجة. يتم اكتشافها تلقائياً بواسطة الوكلاء.
- **ذاكرة وكيل دائمة** — تتذكر الوكلاء عبر الجلسات مع بحث متجهي، وملفات مساحة عمل، وتعلّمات من الأخطاء. تُدار السياق تلقائياً.

## المُدارة مقابل المستضافة ذاتياً: متى يكون كل منهما منطقياً

التمييز بين أطر وكلاء الذكاء الاصطناعي ومنصات وكلاء الذكاء الاصطناعي يهم أكثر في وقت النشر. يمنحك الإطار لبنات بناء — تعريفات الوكلاء، تكاملات الأدوات، أنماط المحادثة. تمنحك المنصة طبقة الإنتاج: أين تعمل الوكلاء، كيف تصل إلى بيانات الاعتماد، ما الذي يوقفها من الخروج عن السيطرة.

**الأطر المستضافة ذاتياً** (LangGraph، CrewAI، AutoGen) تمنحك أقصى تحكم. أنت تمتلك البنية التحتية. أنت تهيّئ الحاويات. أنت تبني خط أنابيب بيانات الاعتماد. يعمل هذا عندما يكون لدى فريقك قدرة DevOps مكرّسة وبنية تحتية قائمة تحتاج الوكلاء للتكامل معها بعمق.

**منصات وكلاء الذكاء الاصطناعي المُدارة** تتعامل مع الطبقة التشغيلية بحيث يركّز فريقك على منطق الوكيل. تقع OpenLegion هنا — ولكن مع فرق حاسم: متوفر المصدر تحت BSL 1.1. تحصل على عمليات بمستوى منصة (عزل، تخزين، ضوابط ميزانية) دون قفل البائع على جانب البنية التحتية.

السؤال ليس أيهما "أفضل". إنه ما إذا كان يجب على فريقك أن ينفق ساعات هندسية على البنية التحتية لأمان الوكلاء أم على الوكلاء أنفسهم.

### متى تكون الاستضافة الذاتية منطقية

- لديك متطلبات صارمة لإقامة البيانات تستبعد أي خدمة مُدارة
- تحتاج وكلاؤك إلى تكامل عميق مع البنية التحتية الموجودة محلياً
- يشغّل فريقك بالفعل مجموعات Kubernetes ولديه ممارسات DevOps ناضجة
- تحتاج إلى تخصيص بيئة وقت التشغيل على مستوى لا تكشفه المنصات المُدارة

### متى تكون منصة وكلاء الذكاء الاصطناعي المُدارة منطقية

- تحتاج إلى وكلاء في الإنتاج في غضون أيام، وليس أشهر
- فريقك من 1-5 مهندسين ولا يمكنه تكريس عدد رؤوس للبنية التحتية
- تحتاج إلى ضمانات [أمان وكلاء الذكاء الاصطناعي](/learn/ai-agent-security) دون بنائها بنفسك
- تريد ضوابط تكلفة وتتبع طلبات دون قياس كل شيء يدوياً

## نموذج مفاتيح API الخاصة بك — لماذا يهم

تفرض معظم منصات الذكاء الاصطناعي المُدارة رسوماً لكل رمز أو تأخذ هامشاً على استخدام النموذج. يخلق هذا مشكلتين: عدم شفافية التكلفة وقفل المزوّد.

تتخذ OpenLegion نهجاً مختلفاً. أحضر مفاتيح LLM API الخاصة بك من أي مزوّد — OpenAI أو Anthropic أو Google أو Mistral أو أي من أكثر من 100 مزوّد مدعوم عبر LiteLLM. تتدفق رموزك مباشرة إلى المزوّد بأسعاره المعلنة. تفرض OpenLegion رسوماً على المنصة والتنسيق، وليس على الوصول إلى النموذج.

يهم هذا لثلاثة أسباب:

**شفافية التكلفة.** ترى بالضبط ما ينفقه كل وكيل على كل مزوّد. لا هامش ربح مخفي. لا "رموز منصة" تحجب التكاليف الحقيقية.

**مرونة المزوّد.** بدّل النماذج لكل وكيل. شغّل GPT-4o لمهام الاستدلال المعقّدة، وClaude لمهام السياق الطويل، ونموذج Llama محلي للتصنيف عالي الحجم — كل ذلك في نفس المشروع، مُدار من نفس لوحة التحكم.

**لا قفل.** إذا غادرت OpenLegion، تذهب مفاتيح API وتهيئات النماذج الخاصة بك معك. لا توجد طبقة نموذج احتكارية للترحيل منها.

## لمن هي

### مطوّرون مستقلون يبنون منتجات وكلاء

أنت تطلق منتجاً مدعوماً بالوكلاء وتحتاج إلى أن يكون آمناً منذ اليوم الأول. تمنحك OpenLegion بنية تحتية إنتاجية — عزل الحاويات، وتخزين بيانات الاعتماد، وضوابط التكلفة — دون توظيف فريق DevOps. ابدأ بقالب فريق مدمج (فريق تطوير، خط مبيعات، Content Studio) وخصّص من هناك.

### فرق الشركات الناشئة التي تطلق بسرعة

فريقك من 2-10 مهندسين. تحتاج إلى وكلاء في الإنتاج في هذا الـ sprint، وليس الربع التالي. التثبيت ثلاثة أوامر: `git clone`، `./install.sh`، `openlegion start`. يهيّئ معالج الإعداد الموجَّه مفاتيح API الخاصة بك، ويختار قالب فريق، وينشر أسطول وكلائك الأول في أقل من ثلاث دقائق.

### فرق أمان المؤسسات

تحتاج إلى تتبع الطلبات ورصد سير العمل، وعزل بيانات اعتماد ينجو من وكيل مخترق، وضوابط ميزانية تمنع التكاليف الجامحة. صُمّمت بنية OpenLegion للبيئات التي تتطلب دفاعاً متعمّقاً. يعني تنسيق نموذج الأسطول القابل للتدقيق أن كل خطوة من سير العمل صريحة وقابلة للتتبع — لا اتخاذ قرار غامض من LLM في مستوى التحكم. راجع صفحة [أمان وكلاء الذكاء الاصطناعي](/learn/ai-agent-security) للحصول على نموذج التهديد الكامل.

## الجاهزية للإنتاج: ما الذي يتعامل معه OpenLegion مقابل افعلها بنفسك

| القدرة | افعلها بنفسك (إطار فقط) | OpenLegion |
|---|---|---|
| **بيئة تشغيل الوكيل** | تهيّئ Docker، وتدير الصور، وتتعامل مع الشبكات | كل وكيل يُوفَّر تلقائياً في حاوية معزولة (384 ميغابايت ذاكرة، 0.15 CPU افتراضياً، غير الجذر، no-new-privileges) |
| **إدارة بيانات الاعتماد** | متغيرات بيئة أو تكامل خزنة مخصّص | وكيل خزنة مع حقن أعمى — الوكلاء لا يرون المفاتيح الخام أبداً |
| **ضوابط التكلفة** | تتبع يدوي، لا حدود صارمة | ميزانيات يومية/شهرية لكل وكيل مع قطع تلقائي |
| **التنسيق** | برمج منطق التوجيه الخاص بك أو استخدم توجيهاً قائماً على LLM | تنسيق نموذج الأسطول (سبورة + نشر/اشتراك + تسليم) — قابل للتدقيق |
| **الرصد** | ادمج LangSmith، Datadog، أو تسجيل مخصّص | لوحة تحكم مدمجة مع بث مباشر، ومخططات تكلفة، وآثار طلبات |
| **النشر متعدد القنوات** | ابنِ تكاملات لكل قناة | CLI، Telegram، Discord، Slack، WhatsApp — بالإضافة إلى نقاط نهاية webhook للتكاملات الخارجية |
| **أتمتة المتصفح** | هيّئ Playwright/Puppeteer، وأدر مثيلات Chrome | Camoufox (Firefox شبحي) لكل وكيل في حاوية خدمة متصفح مشتركة، مع KasmVNC (المنافذ 6100..6163)، وتحكم CDP، واسترداد تلقائي |
| **قابلية توسع الأدوات** | ابنِ تكاملات مخصّصة أو استخدم أدوات LangChain | متوافق مع MCP — اتصل بأي خادم MCP + 50+ مهارة مدمجة، يتم اكتشافها تلقائياً |
| **ذاكرة الوكيل** | ابنِ RAG مخصّص أو إدارة حالة | ذاكرة متجهية دائمة لكل وكيل مع إدارة سياق تلقائية |
| **التعافي من فشل النموذج** | منطق إعادة محاولة مخصّص لكل مزوّد | سلاسل تعافٍ قابلة للتهيئة عبر المزوّدين عبر LiteLLM |

الملخّص: إذا كنت تقيّم [أطر وكلاء الذكاء الاصطناعي](/learn/ai-agent-frameworks) ووجدت نفسك تبني بنية تحتية أكثر من منطق الوكيل، فأنت تحلّ مشكلة منصة بأدوات إطار. تتعامل OpenLegion مع طبقة المنصة بحيث يمكنك التركيز على ما تفعله وكلاؤك فعلاً.

## قابلية توسع الأدوات المتوافقة مع MCP

يدعم OpenLegion بروتوكول Model Context Protocol (MCP) لتوصيل أدوات خارجية. يمكن إضافة أي خادم MCP — قواعد بيانات، أنظمة ملفات، APIs، خدمات داخلية — عبر التهيئة واكتشافه تلقائياً بواسطة الوكلاء. يقع هذا إلى جانب 50+ مهارة مدمجة تغطي أتمتة المتصفح، وعمليات الملفات، وطلبات HTTP، والبحث على الويب، وإدارة الذاكرة، وتنفيذ الشيفرة، والاتصال بـ mesh.

يعني تكامل MCP أن الوكلاء ليسوا محدودين بالقدرات المدمجة. اتصل بخادم Postgres، أو تكامل GitHub، أو API داخلية مخصّصة — تكتشف الوكلاء الأدوات المتاحة تلقائياً وتستخدمها ضمن حدود أذوناتها.

## ذاكرة وكيل دائمة

تحافظ الوكلاء في OpenLegion على الذاكرة عبر الجلسات باستخدام البحث المتجهي، وملفات مساحة العمل، وتعلّمات الأخطاء. عندما يواجه الوكيل مشكلة ويحلّها، يُخزَّن الحل ويُستدعى في جلسات مستقبلية — مما يقلل من الإخفاقات المتكررة ويحسّن جودة التنفيذ مع الوقت.

الذاكرة محصورة لكل وكيل ومخزّنة في قاعدة بيانات SQLite + متجهية المعزولة لكل وكيل داخل حاويته. تحافظ إدارة السياق التلقائية على استخدام الرموز بكفاءة من خلال إظهار الذكريات ذات الصلة فقط للمهمة الحالية، بدلاً من تحميل تاريخ المحادثة بأكمله.

## البنية المعمارية: نموذج الثقة رباعي المناطق

يفصل OpenLegion كل نشر إلى أربع مناطق ثقة بالإضافة إلى مستوى مشغّل أو داخلي:

**المنطقة 0 — مدخلات خارجية غير موثوقة.** أي شيء يصل من المستخدمين أو الأطراف الثالثة: CLI، Telegram، Discord، Slack، WhatsApp، ونقاط نهاية webhook. تُتحقق جميع المدخلات وتُعقَّم عبر حراس الحقن الموجّه قبل الوصول إلى mesh.

**المنطقة 1 — حاويات وكلاء معزولة (غير موثوقة).** يعمل كل وكيل كمثيل FastAPI الخاص به في حاوية Docker مخصّصة بمجلد `/data` خاص به، وقاعدة بيانات ذاكرة، وحدود موارد صارمة. حتى وكيل مخترق بالكامل لا يمكنه الوصول إلى مفاتيح API الخاصة بك، أو بيانات الوكلاء الآخرين، أو نظام المضيف.

**المنطقة 2 — Mesh Host (موثوقة).** خادم FastAPI الذي يشغّل السبورة (حالة مشتركة عبر SQLite + WAL)، وموجّه رسائل PubSub، وخزنة بيانات الاعتماد (الوكيل الذي يتعامل مع الحقن الأعمى)، ومصفوفة ACL، ومدير الحاويات، ومتعقب التكلفة، وخدمة المتصفح (Camoufox لكل وكيل على :8500). هذا هو الدماغ — وهو المكوّن الوحيد الذي يلامس مفاتيح API الخاصة بك.

**المنطقة 2.5 — مشغّل أو داخلية.** عمليات مستوى التحكم المحجوزة المتاحة لوكيل المشغّل أو أدوات mesh الداخلية — إدارة الأسطول، تحرير الوكلاء، منح الأذونات (لا يمكن للمشغّل منح `can_spawn` أو `can_use_wallet`).

**المنطقة 3 — Loopback داخلية فقط.** المستوى الأكثر تقييداً: نقاط نهاية تتطلب ترويسة `x-mesh-internal: 1` وعنوان IP مصدر loopback. تُستخدم لاستدعاءات تنسيق mesh الداخلية فقط.

تعني هذه البنية أن [تنسيق وكلاء الذكاء الاصطناعي](/learn/ai-agent-orchestration) والأمان ليسا اهتمامين منفصلين — إنهما نفس النظام.

## الأسئلة المتكررة

### ما هي منصة وكلاء الذكاء الاصطناعي؟

منصة وكلاء الذكاء الاصطناعي هي بنية تحتية مُدارة تتعامل مع الاهتمامات التشغيلية لتشغيل وكلاء ذكاء اصطناعي مستقلين: عزل الحاويات، وإدارة بيانات الاعتماد، وضوابط التكلفة، والتنسيق، والرصد. تقع فوق أطر مثل LangGraph أو CrewAI وتوفّر طبقة الإنتاج التي تتركها الأطر لك.

### ما أفضل منصة وكلاء ذكاء اصطناعي للإنتاج؟

أفضل منصة وكلاء ذكاء اصطناعي للإنتاج يعتمد على متطلبات الأمان والتشغيل لديك. إذا كنت بحاجة إلى عزل حاويات مدمج، وبيانات اعتماد ممرّرة عبر وكيل الخزنة، وضوابط ميزانية لكل وكيل دون بناء بنية تحتية مخصّصة، يوفّر OpenLegion هذه جاهزة. للفرق المستثمرة بعمق في نظام Microsoft البيئي، يستحق Azure AI Agent Service التقييم. لأقصى مرونة مع المزيد من جهد افعلها بنفسك، تمنحك استضافة LangGraph ذاتياً مع LangSmith رصداً قوياً.

### ما هي منصة وكلاء الذكاء الاصطناعي للمؤسسات؟

تضيف منصة وكلاء الذكاء الاصطناعي للمؤسسات الحوكمة والامتثال وضوابط الأمان فوق تنسيق الوكلاء الأساسي. تشمل المتطلبات الرئيسية: عزل بيانات الاعتماد (يجب ألا يرى الوكلاء مفاتيح API الخام أبداً)، وقابلية تتبع سير العمل، وفرض الميزانية لمنع التكاليف الجامحة، والتحكم في الوصول القائم على الأدوار، وخيارات نشر تدعم متطلبات إقامة البيانات. صُمّمت بنية OpenLegion للبيئات التي تتطلب هذه الضوابط.

### هل يمكنني استضافة وكلاء ذكاء اصطناعي بمفاتيح API الخاصة بي؟

نعم. تستخدم OpenLegion نموذج مفتاح API BYO (أحضر مفتاحك الخاص). تربط مفاتيحك الخاصة من أي مزوّد LLM — OpenAI، Anthropic، Google، Mistral، و100+ آخرين عبر LiteLLM. تتدفق رموزك مباشرة إلى المزوّد بأسعاره المعلنة. أحضر مفاتيح LLM API الخاصة بك. لا هامش ربح على استخدام النموذج.

### مُدارة مقابل مستضافة ذاتياً لوكلاء الذكاء الاصطناعي: ما الفرق؟

تتعامل منصات وكلاء الذكاء الاصطناعي المُدارة مع توفير الحاويات، وتخزين بيانات الاعتماد، وضوابط التكلفة، والرصد لك. الاستضافة الذاتية تعني نشر إطار (LangGraph، CrewAI، AutoGen) على بنيتك التحتية وبناء هذه الطبقات التشغيلية بنفسك. المُدارة أسرع للإنتاج وتتطلب استثمار DevOps أقل. الاستضافة الذاتية تمنح أقصى تحكم في البنية التحتية. تقدّم OpenLegion هجيناً: شيفرة متوفرة المصدر (BSL 1.1) يمكنك استضافتها ذاتياً، مع قدرات منصة مُدارة مدمجة.

### كيف يقارَن OpenLegion بمنصات وكلاء الذكاء الاصطناعي الأخرى؟

يتميّز OpenLegion ببنية تضع الأمان أولاً. بناءً على التوثيق العام في وقت الكتابة، لا يوفّر أي [إطار وكلاء ذكاء اصطناعي](/learn/ai-agent-frameworks) رئيسي آخر بيانات اعتماد ممرّرة عبر وكيل الخزنة مدمجة، أو عزل حاوية إلزامي لكل وكيل، أو فرض ميزانية أصلي لكل وكيل. راجع [مقارنة الأطر](/learn/ai-agent-frameworks) للتفصيل عبر OpenClaw وLangGraph وCrewAI وAutoGen وSemantic Kernel.

### ما الترخيص الذي يستخدمه OpenLegion؟

OpenLegion متوفر المصدر تحت ترخيص BSL 1.1 ومتاح على [GitHub](https://github.com/openlegion-ai/openlegion). يقدّم المشروع أيضاً منصة مستضافة للفرق التي تريد بنية تحتية مُدارة دون استضافة ذاتية.

### ما مدى سرعة نشر وكيلي الأول؟

ثلاثة أوامر وأقل من ثلاث دقائق. `git clone`، `./install.sh`، `openlegion start`. يهيّئ معالج الإعداد الموجَّه مفاتيح API الخاصة بك، ويختار قالب فريق، ويوفّر أسطول الوكلاء المعزول الأول تلقائياً.
